# Chat Redesign - 架构总览

## 设计目标

基于 `SESSION_TOPIC_DESIGN.md` 规范，重构会话页面为三栏布局，支持 AI 分页模式和 PM/Group 连续模式。

---

## 布局架构

### 三栏布局

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Header (可选)                                                           │
├──────────────┬────────────────────────────────┬─────────────────────────┤
│              │                                │                         │
│   Session    │        Message Area            │     Topic Selector      │
│    List      │                                │     (可折叠)             │
│              │  ┌──────────────────────────┐  │                         │
│  ┌────────┐  │  │                          │  │  ┌─────────────────┐    │
│  │Session │  │  │     Message List         │  │  │  Topic 1        │    │
│  │ Item 1 │  │  │                          │  │  ├─────────────────┤    │
│  ├────────┤  │  │                          │  │  │  Topic 2 (当前) │    │
│  │Session │  │  │                          │  │  ├─────────────────┤    │
│  │ Item 2 │  │  └──────────────────────────┘  │  │  Topic 3        │    │
│  ├────────┤  │  ┌──────────────────────────┐  │  ├─────────────────┤    │
│  │  ...   │  │  │    Message Input         │  │  │  + 新建话题      │    │
│  └────────┘  │  └──────────────────────────┘  │  └─────────────────┘    │
│              │                                │                         │
├──────────────┴────────────────────────────────┴─────────────────────────┤
│  Footer (可选)                                                           │
└─────────────────────────────────────────────────────────────────────────┘
```

### 响应式策略：抽屉模式

**断点定义**：
- `lg` (1024px+): 三栏完整展示
- `md` (768px-1023px): 左栏抽屉，中间+右栏展示
- `sm` (<768px): 左栏和右栏都变成抽屉，只显示中间消息区

**抽屉行为**：
- 左侧抽屉：点击汉堡菜单按钮展开会话列表
- 右侧抽屉：点击话题图标按钮展开话题选择器
- 抽屉可以通过点击遮罩层或滑动关闭

## 显示模式

### AI 类型 - 分页模式

每次只显示一个话题的消息，通过滚轮边界触发或点击话题切换。

```
┌─────────────────────────────────────────┐
│  ⬆️ 上一话题提示区 (可选)                │
├─────────────────────────────────────────┤
│                                         │
│        当前话题: "讨论 React 性能优化"    │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │ User: 怎么优化 React 渲染性能？  │    │
│  └─────────────────────────────────┘    │
│  ┌─────────────────────────────────┐    │
│  │ Assistant: 可以从以下几个方面... │    │
│  └─────────────────────────────────┘    │
│                                         │
├─────────────────────────────────────────┤
│  ⬇️ 下一话题/新建话题 提示区              │
└─────────────────────────────────────────┘
```

**滚轮切换逻辑（边界触发）**：
1. 消息列表滚动到顶部边界
2. 用户继续向上滚动（滚轮 delta > 阈值）
3. 触发切换到上一话题
4. 底部同理：滚到底部继续下滚 → 下一话题/新建话题

**切换动画**：
- 使用 CSS transition 或 Framer Motion 实现平滑过渡
- 可选：卡片翻转/滑动效果

### PM/Group 类型 - 连续模式

所有消息连续展示，话题切换处显示分隔线。

```
┌─────────────────────────────────────────┐
│  消息 1 (话题 A)                         │
│  消息 2 (话题 A)                         │
├───────── 话题切换: 新主题 B ─────────────┤
│  消息 3 (话题 B)                         │
│  消息 4 (话题 B)                         │
├───────── 话题切换: 讨论 C ──────────────┤
│  消息 5 (话题 C)                         │
└─────────────────────────────────────────┘
```

**分隔线逻辑**：
- 渲染消息列表时检测相邻消息的 `topic_id`
- 当 `topic_id` 变化时插入分隔线组件
- 分隔线显示话题名称，可点击跳转到话题开始位置

**话题选择器交互**：
- 点击话题 → 平滑滚动到该话题的第一条消息位置
- 使用 `scrollIntoView` 或自定义滚动逻辑

## 技术栈

### UI 组件库

| 组件类型 | 使用库 | 说明 |
|---------|-------|------|
| 基础组件 | shadcn/ui | Button, Card, Dialog, Input 等 |
| 消息渲染 | @lobehub/ui | ChatItem, Markdown, Avatar |
| 布局 | shadcn/ui + Tailwind | 响应式三栏布局 |
| 动画 | Framer Motion (可选) | 话题切换动画 |

### 新增依赖

```json
{
  "@lobehub/ui": "^1.x.x",
  "framer-motion": "^11.x.x" // 可选
}
```

### lobeui 组件选用

- **ChatItem**: 消息气泡组件，支持 Markdown、代码高亮
- **Markdown**: 富文本渲染，支持数学公式、代码块
- **Avatar**: 头像组件，支持用户/AI 区分
- **ActionsBar**: 消息操作栏（复制、删除等）
