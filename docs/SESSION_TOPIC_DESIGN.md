# Session-Topic-Message 设计规范

## 数据模型

### 层级关系
```
Session (会话)
  └── Topic (话题) ─ 可选
        └── Message (消息)
```

- 一个 Session 包含多个 Topic
- 一个 Topic 包含多个 Message
- Message 的 `topic_id` 可以为 null（孤立消息）

### Session 类型

| 类型 | 说明 | 交互模式 |
|------|------|----------|
| `ai` | AI 对话 | 分页：每次显示一个话题 |
| `pm` | 私聊 | 连续：所有消息连续滚动 |
| `group` | 群聊 | 连续：所有消息连续滚动 |

## API 规范

> 完整定义见 `samples/api_openapi.json`

### 关键特性

| 特性 | 支持 | 说明 |
|------|------|------|
| Session.type = 'ai' | ✅ | 类型: `pm/group/ai/<plugin_type>` |
| Message.topic_id = null | ✅ | 消息可以不属于任何话题 |
| 按 Session 获取消息 | ✅ | 不传 topic_id 返回所有消息 |

### 主要端点

```
GET  /api/v1/sessions/{id}/messages           获取消息（可选 topic_id 筛选）
GET  /api/v1/sessions/{id}/topics             获取话题列表
POST /api/v1/sessions/{id}/messages           创建消息
POST /api/v1/sessions/{id}/topics             创建话题
POST /api/v1/sync                             批量同步
```

### 获取消息参数

```
GET /api/v1/sessions/{session_id}/messages

参数:
- topic_id (可选): 不传返回所有消息，传值只返回该话题消息
- page: 页码
- size: 每页数量
```

## 核心设计

### 1. 话题创建时机

**变更**: 从「创建 Session 时自动创建话题」改为「发送首条消息时创建话题」

```
创建 Session → 无默认话题
发送首条消息 → 自动创建话题 → 消息关联到该话题
```

好处：
- 避免空话题
- 话题名称可基于首条消息内容生成

### 2. 孤立消息处理

孤立消息 = `topic_id` 为 null 的消息

**处理策略**:
1. 按时间间隔分组（建议 30 分钟）
2. 每组创建一个「虚拟话题」
3. 虚拟话题仅存本地，不同步到服务器

```
孤立消息 [M1, M2, M3] (间隔 < 30min)
    → 虚拟话题 "历史对话 2024-01-15"
    → M1, M2, M3 关联到此话题
```

### 3. 虚拟话题

| 属性 | 说明 |
|------|------|
| `is_virtual = true` | 本地字段，标记为虚拟 |
| 不同步 | 创建/更新不调用 API |
| 消息同步 | 该话题下的消息，`topic_id` 设为 null 后同步 |

## 显示模式

### AI 类型 - 分页模式

每次只显示一个话题的消息，通过手势或导航切换话题。

```
┌─────────────────────┐
│ ↑ 上拉切换上一话题   │
├─────────────────────┤
│                     │
│   当前话题的消息     │
│                     │
├─────────────────────┤
│ ↓ 下拉切换下一话题   │
│   或新建话题         │
└─────────────────────┘
```

**切换逻辑**:
- 滚动到顶部边界 → 切换到上一个话题
- 滚动到底部边界 → 切换到下一个话题，或新建话题
- 建议：边界处有阻力效果和反馈

### PM/Group 类型 - 连续模式

显示所有消息，话题切换处插入分隔线。

```
┌─────────────────────┐
│ 消息 1              │
│ 消息 2              │
├─────────────────────┤
│ ── 话题: 新主题 ──   │  ← 分隔线
├─────────────────────┤
│ 消息 3              │
│ 消息 4              │
└─────────────────────┘
```

**分隔线逻辑**:
- 检测相邻消息的 `topic_id` 是否变化
- 变化时插入分隔线，显示话题名称

## 本地扩展字段

以下字段仅存本地，**不通过 API 同步**:

| 实体 | 字段 | 用途 |
|------|------|------|
| Session | `current_topic_index` | 记录当前显示的话题索引 |
| Topic | `is_virtual` | 标记虚拟话题 |
| Topic | `order_index` | 本地排序 |

---

## 同步规则

### Session / Message
正常同步，无特殊处理。

### Topic
```
if topic.is_virtual:
    跳过，不同步
else:
    正常同步
```

### 虚拟话题下的 Message
```
同步时: topic_id = null
```

### 从服务器拉取后
```
检测 topic_id = null 的消息 → 执行孤立消息处理 → 分配到虚拟话题
```
