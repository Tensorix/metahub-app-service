# 功能特性详解

## 🎨 UI/UX 设计亮点

### 1. 侧边栏设计（重点功能）

#### 展开状态
- **宽度**: 256px (w-64)
- **顶部**: 应用 Logo "MetaHub" + 折叠按钮
- **用户区**: 圆形头像（显示用户名首字母）+ 用户名 + 邮箱
- **导航菜单**: 
  - 首页（Home 图标）
  - 会话（MessageSquare 图标）
  - 设置（Settings 图标）
- **底部**: 主题切换按钮 + 退出登录按钮

#### 折叠状态
- **宽度**: 64px (w-16)
- **仅显示图标**: 所有文字隐藏
- **居中对齐**: 图标居中显示
- **保持功能**: 所有功能正常可用

#### 动画效果
- **平滑过渡**: `transition-all duration-300`
- **图标旋转**: 折叠按钮图标切换动画
- **悬停效果**: 菜单项悬停高亮

#### 交互细节
- **当前页面高亮**: 使用 `secondary` 变体
- **悬停反馈**: Ghost 按钮悬停效果
- **点击反馈**: 按钮按下效果

### 2. 主题系统

#### 三种模式
1. **浅色模式** (Light)
   - 白色背景
   - 深色文字
   - 清爽明亮

2. **深色模式** (Dark)
   - 深色背景
   - 浅色文字
   - 护眼舒适

3. **跟随系统** (System)
   - 自动检测系统主题
   - 实时响应系统变化

#### 实现细节
- **CSS 变量**: 使用 Tailwind CSS v4 的主题变量
- **类名切换**: 根元素添加 `.dark` 类
- **持久化**: localStorage 保存用户选择
- **初始化**: 页面加载时自动应用保存的主题

### 3. 登录/注册页面

#### 设计特点
- **居中布局**: 卡片式设计，屏幕居中
- **响应式**: 移动端友好
- **表单验证**: 实时验证和错误提示
- **加载状态**: 提交时显示加载动画
- **错误处理**: 友好的错误提示

#### 注册表单
- 用户名（必填，至少 3 个字符）
- 邮箱（可选，格式验证）
- 手机号（可选）
- 密码（必填，强度要求）
- 确认密码（必填，一致性验证）

#### 密码强度要求
- 至少 8 个字符
- 包含大写字母
- 包含小写字母
- 包含数字
- 包含特殊字符

### 4. 首页设计

#### 欢迎信息
- 大标题: "Hi, {username} 👋"
- 副标题: "欢迎回到 MetaHub"

#### 信息卡片
1. **账户信息卡片**
   - 用户名
   - 邮箱
   - 手机号
   - 账户状态

2. **快速开始卡片**

## 🔐 安全特性

### 1. 密码处理
- **前端哈希**: 使用 Web Crypto API 进行 SHA256 哈希
- **不传明文**: 密码在发送前已哈希
- **后端二次哈希**: 后端使用 bcrypt 再次哈希

### 2. Token 管理
- **Access Token**: 短期有效（默认 30 分钟）
- **Refresh Token**: 长期有效（默认 7 天）
- **自动刷新**: Access Token 过期时自动刷新
- **安全存储**: 使用 localStorage 存储

### 3. 路由保护
- **ProtectedRoute**: 未登录自动跳转登录页
- **加载状态**: 验证期间显示加载动画
- **自动初始化**: 页面加载时验证登录状态

## 🔄 状态流转

### 登录流程
```
用户输入 → SHA256 哈希 → 发送请求 → 接收 Token → 
保存 Token → 获取用户信息 → 更新状态 → 跳转首页
```

### Token 刷新流程
```
请求失败 (401) → 检查 Refresh Token → 发送刷新请求 → 
接收新 Token → 更新 Token → 重试原请求
```

### 主题切换流程
```
用户点击 → 更新状态 → 保存到 localStorage → 
更新 DOM 类名 → CSS 变量生效 → 界面更新
```

## 🎨 设计系统

### 颜色方案
- **Primary**: 主要操作按钮
- **Secondary**: 次要操作、当前选中
- **Muted**: 辅助文字、禁用状态
- **Destructive**: 危险操作（如退出登录）
- **Border**: 边框、分隔线

### 间距系统
- 使用 Tailwind 的间距系统
- 保持一致的间距比例
- 响应式间距调整

### 圆角
- 统一使用 `--radius` 变量
- 可全局调整圆角大小

## 🛠️ 可扩展性

### 添加新功能
1. 在 `pages/` 添加新页面
2. 在 `App.tsx` 添加路由
3. 在 `Sidebar.tsx` 添加菜单项
4. 在 `lib/api.ts` 添加 API 方法

### 添加新组件
```bash
npx shadcn@latest add [component-name]
```

### 添加新状态
在 `store/` 创建新的 Zustand store

## 📊 技术选型理由

### React 19
- 最新特性
- 性能优化
- 更好的并发渲染

### Vite 7
- 极速冷启动
- 即时热更新
- 优化的构建

### shadcn/ui
- 不是 npm 包，而是复制代码
- 完全可定制
- 基于 Radix UI（无障碍访问）
- Tailwind CSS 样式

### Zustand
- 轻量级（< 1KB）
- 简单易用
- 无需 Provider
- TypeScript 友好

### Tailwind CSS v4
- 最新版本
- 更好的性能
- 原生 CSS 变量
- 更小的包体积
